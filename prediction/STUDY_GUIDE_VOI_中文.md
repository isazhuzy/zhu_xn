# 学习指南 —— VOI / OIR / MPB：一个真正有效的多因子盘口预测
### 基于 D. Shen (2015), *Order Imbalance Based Strategy in High Frequency Trading*（高频交易中的订单失衡策略），牛津大学数学研究所硕士论文。

本系列第 3 篇，也是离我们最近的一篇：Shen 用的是 **CFFEX 沪深300指数期货（IF）、500ms 快照、2014 年 244 个交易日**——和我们的 `hft_future_ts` 是完全相同的市场和数据节奏，只比我们的样本早了六年。代码：`voi_ddb.py` / `voi_plot.py`。

## 0. 一句话核心思想
> **来自最近几张快照的三个廉价数字——买卖最优队列如何*变化*（VOI）、它们有多*不平衡*（OIR）、以及成交打在相对中间价的哪一侧（MPB）——线性地预测未来数秒的平均中间价，且样本外成立。**

第 1 篇（OFI）证明了订单流*解释*同一区间的价格变动。这一篇跨过了那条线，进入真正的*预测*——同时也带着预测该有的、诚实的 R² 下降。

## 1. 三个因子（全部可从单档 L1 数据算出）

**符号约定：**
- `pb` = 买一价，`pa` = 卖一价 
- `qb` = 买一量，`qa` = 卖一量
- `M` = 中间价 = (pb + pa) / 2；`s` = 价差 = (pa − pb)，以 tick 计
- 数据字段：`m_nBidPrice/m_nAskPrice`→pb/pa，`m_nBidVolume/m_nAskVolume`→qb/qa

### 1.1 VOI —— 盘口增量失衡（Volume Order Imbalance）
相邻两张快照之间，逐边计算：
```
            ⎧ 0             若 买一价 下降    （队伍死了/撤退 —— 记 0）
ΔV_bid  =   ⎨ qb − qb_上一张  若 买一价 不变    （净增加 = 新挂单减去撤单/成交）
            ⎩ qb            若 买一价 上升    （在更高价位重建的全新队伍）
ΔV_ask  =   镜像（涨 ↔ 跌 互换）
VOI     =   ΔV_bid − ΔV_ask
```
VOI 为正 = 买方在增强。

### 1.2 OIR —— 盘口静态失衡（Order Imbalance Ratio）
```
OIR = (qb − qa)/(qb + qa)
```

### 1.3 MPB —— 成交价相对中间价的偏离（Mid-Price Basis）
```
TP_t  = Δ成交额 / (Δ成交量 × 合约乘数)          ← 这半秒所有成交的成交量加权均价(VWAP)
        （该区间无成交时，用上一个有成交区间的值前向填充）
MPB_t = TP_t − (M_t + M_{t−1})/2               ← 减去这半秒的“平均中间价”
```
成交实际打在价差内的哪一侧？
买方抬价吃卖一 → TP 推向卖一 → MPB > 0 = 主动买入。
MPB 是*成交侧*信息（类似第 1 篇的 TI），打包成价格而非成交量。

**逐块拆解（为什么这样定义）：**
1. **平均成交价 TP_t ** `m_iVolume`（这半秒成交多少手）和 `m_iTurnover`（这半秒成交多少钱）都是**增量**字段（每张快照单独的量），不是累计值（累计值是 `m_iAccVolume`/`m_iAccTurnover`）。成交额 ÷（成交量 × 乘数）= 这半秒所有成交的 VWAP。例（IF，乘数 300）：成交 10 手、成交额 11,703,000 元 → TP = 11,703,000 ÷ (10×300) = 3901.0。
2. **为什么减两张中间价的平均，而不是当前 M_t。** 成交发生在 **[t−1, t] 这半秒区间内**，M_{t−1}、M_t 是区间的起点和终点。拿区间内的成交价去比区间的**平均**中间价 (M_t+M_{t−1})/2，时间对齐更准；只比终点会拿“整段成交”对“某一瞬报价”，对不齐。
3. **正负号含义。** TP 贴卖一（成交价高）→ MPB > 0 → **主动买**；TP 贴买一 → MPB < 0 → **主动卖**。符号直接给出这半秒成交的 aggressor（发起）方向。
4. **无成交的半秒。** 有些 500ms 一笔未成交，TP 无法算，代码用**前向填充**沿用最近一次成交价——“没有新成交”不代表观点变了。
5. **为什么 MPB 只有当前值、没有滞后（对比 VOI/OIR 各带 5 个滞后）。** VOI/OIR 描述盘口**状态**，状态持续几秒（这半秒队伍厚、下半秒多半仍厚），历史值还带信息、值得留几期；MPB 是一次性的**成交事件**，打完就过去，信息几乎全在当下，几秒前打哪侧与现在无关 → 留一个当前项即可。这是“状态 vs 事件”的直觉。
6. **合约乘数**：IF/IH = 300 元/点，IC/IM = 200 元/点。

## 2. 预测目标：未来平均中间价变动
```
y_k(t) = mean( M_{t+1}, …, M_{t+k} ) − M_t        k ∈ {1, 4, 20, 120} 个快照
                                                    = 0.5s, 2s, 10s, 60s
```
取未来 k 个中间价的*平均*（而非单点值 M_{t+k}），把买卖价差的来回跳动从目标里滤掉。

## 3. 两个模型

**先想清楚“模型”是什么。** 一个模型就是一个配方：把此刻盘口的几个数字**加权求和**，吐出预测值 ŷ（“我猜下半秒中间价动多少 tick”）。难点只在两件事：①放哪些数字进去（因子）②每个数字乘多大权重（系数 β）。“训练模型”就是让电脑从历史里找出让预测最准的那组 β。

**逐层搭起来：**
- **单因子** `ŷ = β0 + β1·VOI`。fig86 左图那条线——β1 是它的**斜率**（IF = 0.054：VOI 每多 1 手净流入，下半秒平均多动 0.054 tick）。样本外 R² ≈ 3.1%（IF）。
- **模型A**（加 VOI 滞后3秒）：不只用此刻 VOI，还用前 5 张快照（≈3 秒）的 VOI，各配自己的 β。动机：订单流有惯性。但 fig86 右图显示柱②≈柱①——相邻快照的 VOI 高度相似，多加的是同一信息的拷贝。**堆同类信息，边际递减。**
- **模型B**（三因子 + 各带滞后 + 除价差）：喂进**三种不同性质**的信息 （因子），R² 从 3% 增长到 8.7%。

```
模型A:  y_k ~ β0 + Σ_{l=0..5} β_l · VOI_{t−l}
模型B:  y_k ~ β0 + Σ β_l·VOI_{t−l}/s_t + Σ γ_l·OIR_{t−l}/s_t + δ·MPB_t/s_t
        │        └─ 流量(6项) ────┘   └─ 水平(6项) ───┘   └ 成交(1项) ┘
        └ 截距；共 1+6+6+1 = 13 个系数
```
- **滞后 0..5**（≈ 过去 3 秒）：订单流有自相关，前几张快照的流量仍带信息。
- **除以价差 `s_t`**（以 tick 计，Shen 的关键改进）：价差宽时，同样的失衡意义更小。归一化让一套系数适配所有价差状态。
- 全程都是普通 **OLS**——13 个系数，没有机器学习，完全可解释。
- **B 远胜 A**：“**不同种类**的信息”。VOI 滞后全是挂单意愿且互相重复；OIR 加了“存量水平”、MPB 加了“真实成交方向”。
## 3.5 系数 β 到底怎么算出来的（最小二乘 OLS）

**核心思想：找一组 β，让预测的总误差最小。** 想象 fig86 那条线可画成任何斜率，电脑要选唯一一条——让所有历史点到线的**竖直距离的平方之和**最小的那条。用平方（而非绝对值）的好处：大误差惩罚更重、处处可导、有漂亮的闭式解。

- **单因子**（就是 fig86 斜率的算法）：
  ```
  β1 = Σ(xᵢ−x̄)(yᵢ−ȳ) / Σ(xᵢ−x̄)²  =  Cov(x,y) / Var(x)
  ```
  人话：β1 =（x 与 y 一起变动的程度）÷（x 自己变动的程度）。斜率 0.054 就这么来的。
- **13 个因子**（模型B）：把 13 个因子摆成矩阵 X（每行 = 一个快照的 13 个因子值，每列 = 一个因子），真实未来变动摆成一列 y。最小二乘解是矩阵公式：
  ```
  β̂ = (XᵀX)⁻¹ Xᵀy
  ```
  做的事和单因子一样——在 13 维里同时找到让总平方误差最小的那组权重。
- **怎么在 1.3 亿行上算却不爆内存。** 公式真正需要的只有 `XᵀX`（13×13 矩阵）和 `Xᵀy`（13×1 向量），二者都是**对所有行求和**。于是**逐月**累加每月的 XᵀX、Xᵀy，整段历史从不同时进内存，最后解一个 13×13 小方程得到精确 β。这就是“矩累加 / 充分统计量”技巧（和 fig86 单因子里用 6 个和 n,Σx,Σy,Σxy,Σx²,Σy² 精确算 R² 是同一思路，升到 13 维）。
- **和 R² 的关系。** R² 来自同一次拟合：`R² = 1 − 最小化后的平方误差 ÷ 总平方波动`。“找 β”和“算 R²”是一件事的两面——先找到误差最小的 β，剩下那点误差占总波动的比例，1 减去它就是 R²。
- **系数符号 = 经济含义。** 每个 β 都可读：VOI 的 β 为正 = 买方增力预示上涨（合理）；MPB 的 δ 应为正 = 主动买预示上涨。若某个符号与直觉相反，通常是共线性（因子互相重复）或数据问题的信号，值得查。

## 4. 纪律：训练/测试切分
- **训练 2020-01 … 2024-12** → 估一次 β，然后冻结。
- **测试 2025-01 … 2026-05** → 套用冻结的 β；报告**样本外 R²**：
  `R²_oos = 1 − Σ(y−Xβ_train)² / Σ(y−ȳ_test)²`。
- 实现技巧：我们逐月累加矩块（XᵀX、Xᵀy、yᵀy、n），于是训练期的合并拟合和精确的样本外 R² 都来自充分统计量——无需把上亿行数据放进内存。（`voi_ddb.py`）

记住我们为何如此偏执：我们自己 `ticker/` 的开盘反转在 38 天上样本内 R² 有 0.56，样本外**直接蒸发**。一个信号只有在它从没见过的数据上仍然成立，才算真的。

## 5. 我们的结果（全样本；训练 2020–24，测试 2025–26）

> 权威数字见 `voi_results.csv`、`voi_hitrate.csv`、`voi_permonth.csv`；图 81–83。

要点（训练 n = 每合约 1300–2900 万行，测试 n = 700–900 万行）：
- **它确实预测，且样本外成立。** 模型B 在 k=1（下一个 0.5s），用冻结的 2020–24 系数套 2025–26 数据：样本外 R² = **8.7%（IF）、8.2%（IH）、7.0%（IM）、5.2%（IC）**。对比：第 1 篇里滞后 OFI 的预测 R² 只有约 0.1–1%。正确的因子 + 滞后 + 价差归一，换来了一到两个数量级。
- **R² 随预测期快速衰减**（图81）：约 5–9%（0.5s）→ 3.5–6%（2s）→ 1–1.7%（10s）→ 0.15–0.3%（60s）。盘口知道的是未来几秒，不是未来一分钟。
- **模型B ≫ 模型A**（每个预测期都如此）：加入 OIR + MPB 并除以价差，在 k=1 把 R² 相对纯 VOI 滞后翻了约 3–5 倍（如 IF 样本外 2.8% → 8.7%）。异质信息（流量 + 水平 + 成交）胜过同类信息堆量。
- **方向命中率随信号强度上升**（图83，样本外 k=20）：无条件约 54.5–55.6%，条件于 |ŷ| > 0.5 tick 则达到 **62.5%（IH）、60.5%（IF）、约 58%（IC/IM）**，触发比例为 14–33% 的快照。一个可用的置信度旋钮。
- **跨时间稳定——且样本外不衰减**（图82）：月度 R² 随波动率状态起伏，但从不消失。值得注意的是，2025–26 的样本外 R² *高于*训练期合并 R²（IF 8.7% vs 7.3%）——冻结的系数没老化；2025–26 只是比 2020–24 平均更受订单流驱动的市场。在测试期重新拟合，也只比冻结 β 多约 1 个百分点。
- 跨合约：IF/IH（窄价差）最强，IC 最弱——和第 2 篇同一原因：价差越窄 ⇒ 信号越干净。（IM 样本外偏高有一部分是时期效应：它的测试窗口恰好是它流动性最好的年代。）
- **对比 Shen 自己 2014 的数字**：他报告在 IF 上、k=20（10s），纯 VOI 模型平均 R² ≈ 0.03、扩展模型 ≈ 0.07。我们 IF 在 k=20 的样本内是 0.010（A）/ 0.016（B）——*结构*复现了（B ≈ 2–5 倍 A，形状相同、随期限衰减相同），但强度只有大约**三分之一到一半**。可能原因：2014 是限制前、超高流动性的年代（2015.09 中金所的严管砍掉了成交量、改变了微观结构），此后更多竞争把容易的部分套利掉了。一个"现象仍在、幅度衰减"的完美例子。

## 6. 从预测到损益 —— 用回测量化"信号在价差之内" ⚠️

我们把完整的 Shen 式阈值策略在样本外跑了一遍（`voi_backtest.py`，图 84–85）：冻结的 2020–24 模型B、k=20 系数，2025–26 测试期，|ŷ| > q 时开一手，持有 10 秒，非重叠。每笔按五个越来越接近现实的执行/成本层次分别记账：

| 每笔平均（tick），q=2.0 | IC | IF | IH | IM |
|---|---|---|---|---|
| **中间价 → 中间价（信号价值）** | **+1.79** | **+1.47** | **+1.17** | **+1.90** |
| 跨价差，同一快照 | −1.07 | −0.14 | −0.25 | −0.47 |
| 跨价差，延迟 500ms | −4.49 | −1.86 | −1.62 | −3.25 |
| + 手续费 开0.23/平昨0.23 bp | −6.09 | −2.85 | −2.28 | −4.85 |
| + 手续费 开0.23/平今3.45 bp | −17.3 | −9.77 | −6.86 | −16.0 |

从上往下读——每一行都是现实的一层：
1. **信号价值确实为正，且随门槛升高而增大**（+0.3 → +1.9 tick，t 值 17–80）。预测是有效的。这一行就是把 R² 变得可触摸。
2. **一跨价差就全吃掉。** 即便零延迟、零手续费，最好的情形（IF，q=2）也只有 −0.14 tick——离一个价差来回还差约 0.15–1 tick。这就是"信号住在价差之内"的精确、量化含义。
3. **延迟很残酷：一个 500ms 快照就要付约 0.7–3 tick**（IF −0.14 → −1.86）。预测*立刻*兑现——绝大部分预测的移动发生在第一个快照内。谁更快，谁收走。
4. 手续费只是把尸体埋得更深：平昨加约 1–1.6 tick，平今（3.45bp）加 5–12 tick——单是 2015 年后的平今费率，就让任何 10 秒的来回不可想象。
底线（图85）：q=0.5、现实延迟 + 平昨费率下，这个"策略"每手每天亏 12–25 万元，Sharpe ≈ −80 到 −120。整个门槛扫描里没有任何一处翻正。

所以诚实的变现结论没变，只是现在有了数字：
- 作为 **taker（吃单方）**，你在手续费*之前*就已经差 0.15–1 tick——永远地一到就死，除非你的延迟接近于零、且只吃最强的那些信号（即便如此你也只是边际玩家）。
- 作为 **maker（做市方）**，经济学反转：你*收*价差而非付价差——那个盖不住一次跨价差的 +1.5 tick 信号价值，变成了对你挂单的保护（把预测说要被打穿的那一侧撤掉/收缩）。
- 对我们：一个**叠加在本来就要执行的交易上的短期公允漂移**——进场慢信号时，是坐在买一等，还是抬手吃卖一；每笔 0.3–1.9 tick 正是这种择时能省下的预算。
（Shen 报告 2014 IF 上 Sharpe 约 6–7——但那是在 0.25bp 成本假设、贴价成交、严管前市场下。我们这张表，就是那个世界剩下的东西。）

## 7. 四篇论文如何拼在一起
| 论文 | x → y | 期限 | 在我们数据上的结论 |
|---|---|---|---|
| #1 CKS OFI | 流量 → *同区间* ΔP | 同期 | R² ≈ 0.5 —— 解释，不预测 |
| #2 Gould-Bonart QI | 盘口水平 → *下一次变动方向* | 一 tick 之后 | 命中 59–68% —— 预测方向 |
| #3 Shen VOI+ | 流量 + 水平 + 成交 → 未来 k 秒漂移 | 0.5–60 s | 样本外 R² 高达约 0.11 —— 预测幅度 |
| #4 Stoikov 微观价格 | 盘口水平 → *当下的公允价* | 极短期限的极限 | 比 mid 更准的尺子（见指南4） |

同一个对象——L1 盘口状态及其变化——被读了四种。可预测的成分活在**秒级**期限、**价差以内**的幅度上；每种方法都同意这一点。

## 8. 练习
1. 推导：当按快照差分计算时，VOI =（买方贡献）−（卖方贡献），也就是第 1 篇 OFI 的形式。两者定义究竟在哪里不同（提示：价格相等的情形要处理"持平"）？
2. 对 IF 重算模型B，但 MPB *不*除以价差。δ 在不同价差状态下还稳定吗？
3. 加上滞后 6..20。看着样本内 R² 慢慢爬升、而样本外 R² 不跟——这是过拟合的现场演示。
4. 计算触发规则的损益，毛的与扣 1 tick 的：自己验证第 6 节的算术。
5. Shen 在 2014 IF 上回测了一个阈值策略（|ŷ| 越过某分位数就交易）——在 IM（最宽价差、最难的情形）上复现，并比较扣费后的结果。

## 9. 迷你词汇表
**VOI** 盘口增量失衡（快照流量）· **OIR** 盘口静态失衡（队列水平）· **MPB** 成交价基差（平均成交价 − 平均中间价）· **除价差归一** 把自变量除以以 tick 计的价差 · **y_k** 未来 k 个快照的平均中间价变动 · **样本外 R²** 用冻结系数在未见数据上解释的方差比例 · **矩累加** 用 XᵀX/Xᵀy 的和做合并 OLS · **taker/maker** 跨价差吃单 vs 在队列里挂单等待。
