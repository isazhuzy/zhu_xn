# Tick 数量研究 · 阶段小结（interim）

承接分钟内动量/反转研究（step1/2/3），系统考察「每分钟有多少 tick」以及它对策略收益与曲线可信度的影响。

- **数据**：DolphinDB `dfs://hft_future_ts`，IC/IF/IH/IM，2020-01..2026-05，500ms 快照（**理论上限 120 tick/分钟**），统计按**去重时间戳**（避开 2024-02 的重复行）。
- **可复算 CSV**：`ticks_per_minute_hist.csv`、`intraday_ticks.csv`、`threshold_scan.csv`。

---

## 1. 阈值扫描：提高 tick 阈值能不能提收益？（fig55）
对每分钟只保留 tick 数 `L ≥ T` 的交易，扫描 T。
- **结论：tick 阈值不是收益杠杆。** step3 的 V 型毛利在 T=10→100 几乎是平的（高 t，大样本）。
- **`L≥120` 是陷阱**：120 是 500ms 上限，`L≥120` 只能选到「超密 tick」的特殊月（2022-12 等），样本小、时间扎堆——step1 在那里**符号都翻**（反转→延续 +0.57），step3 edge 直接塌（t 从 6→0.1）。naive 取 max 会误选它。
- **唯一的小甜点**：限制到「接近满 tick 的活跃分钟」`L≈100–110`，step3 有 **+0.01~0.04** 的小幅、稳健提升（IC T110 +0.250 t6.4；IM T100 +0.358 t4.1），但仍远小于价差。

## 2. 每分钟实际 tick 数（fig56 分布 / fig59 均值）
| 合约 | 均值 | 中位数 | ≥115满tick |
|---|---|---|---|
| IM 中证1000 | 105 | 112 | 43% |
| IF 沪深300 | 102 | 107 | 40% |
| IC 中证500 | 101 | 106 | 27% |
| IH 上证50 | 88 | 91 | 9% |

- 分布**左偏**：多数分钟接近满档（120），一条「安静分钟」的尾巴把均值拉到中位数以下。
- **IM 最密、IH 最稀**，跟成交活跃度一致。`MIN_TICKS=10` 几乎不筛掉任何东西（p10 都 ≥34）。

## 3. 日内 tick 活跃度（fig60）
每个半天一个 **U 型**：
- **09:30 开盘 ≈ 119**（四合约几乎都顶满 120，每 500ms 一笔）。
- 上午一路衰减到 11:29（IC/IF/IM ~98–102，**IH 最低 ~85**）。
- 13:00 午后回升，午盘中段再低，**尾盘冲向 15:00 回升**。
- **IH 全天最稀**；整点/半点有小尖峰。（11:30、15:00 收盘单笔已剔除。）

## 4. 对 2 分钟 bar 曲线可信度的影响（fig47 / fig58）
2 分钟 bar 长度不齐（每分钟 tick 数不同），曲线按 tick 对齐取列向 nanmean，**尾部覆盖变薄会失真**。
- fig47 的实线在 **覆盖率 ≥ 90%** 处截断：IC tick155 / IF 158 / IH 123 / IM 159（IH 最早，因为最稀）。
- **能撑到完整 tick 240（两分钟都顶满 120）的 bar 极少**：IM 9.4% / IC 2.7% / IF 2.7% / **IH 0.5%**。
- fig58：用**全部 bar** 画全程，但随覆盖变薄淡化（粗实线≥90% · 细线60–90% · 点线<60%），并在每条线上 **●标出 fig47 的 90% 终点**——既不丢数据，又能看清哪段不可信。

---

## 结论一句话
**典型一分钟 ~100 个 tick（IH ~88，IM ~105），理论封顶 120，开盘满档、日内 U 型；tick 数量本身不是收益来源，提高阈值最多让毛利结构干净一点点，`L≥120` 反而是小样本陷阱；它真正的用处是界定 2 分钟 bar 曲线尾部从哪里开始不可信。**
