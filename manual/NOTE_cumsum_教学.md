## 一、cumsum 是什么

`np.cumsum(x)` = **累计和 / 滚动总计**：输出的第 i 个元素 = x 的前 i+1 个元素之和。

```python
np.cumsum([2, -1, 3])   # -> [2, 1, 4]
```

本质：**把"每一步的增量"变成"走到哪里的水位"** —— 离散版的积分。
- 每笔盈亏 → 资金曲线
- 价格变动 → 价格路径
- 每天降雨 → 水库水位

## 二、为什么有用（统计意义，最核心的一段）

单个 tick 的未来变动 dy 噪声极大：±几个 tick 的噪声包着 ±0.3 tick 的均值，单点信噪比 ≈ 0.1，散点图上什么都看不见。但**噪声和信号的累积速度不同**：

- 噪声正负相抵，n 步后只游走到 **√n** 的量级；
- 偏差（真实信号）每步同向，累积成 **n** 的量级。

所以 100 万个 tick 之后：0.2 tick 的偏差累积成 20 万，噪声只游走到 ±3000。
**cumsum 用 √n 的倍数放大一切系统性倾向** —— 单点看不见的微弱信号，在累计曲线上变成明显的斜坡。这也是为什么看策略要看资金曲线（cumsum），统计学检测微小偏移用 CUSUM 图。

## 三、排序后再 cumsum：多出来的一层含义

我们不按时间序 cumsum，而是**按因子值从小到大排序后**再 cumsum。这时：

> **曲线在任何一点的斜率 = 该因子水平的 tick 的平均未来收益 E[dy | factor]。**

- 左边下行 = 负因子后价格平均下跌；右边上行 = 正因子后平均上涨
- **对勾形（✓）= 因子单调有预测力**；乱走 = 没有预测力
- **最低点 = 条件均值换号处 = 天然的多空阈值**（三个因子都在 ≈0）
- 平台（如 VOI=0 大块）= 该区域因子不含信息
- 曲率 = 效果在尾部是增强还是饱和

回归只给一个斜率 + R²；排序 cumsum 给出**整条函数形状**，不假设线性，还自带 √n 降噪。

**读图两眼法**：
1. **谷底深度 vs 终点** = 信号强度（深度 ÷ n = 每笔毛收益）
2. **终点 vs 0** = 样本整体漂移 ×（约 k 倍窗口重叠），与因子无关 —— 三个因子的终点几乎一样（IC 都约 +8 万），就是这个道理，排序改变不了总和。

## 四、核心代码（就三行）

```python
order = np.argsort(a["voi"].to_numpy(), kind="stable")  # 排序名次（返回下标，不动原数组）
cum   = np.cumsum(a["dy20"].to_numpy()[order])          # 按因子顺序累计未来变动
q     = rank / n                                        # 百分位 = 名次/总数（经验CDF，纯数数）
```

- `argsort` 返回"能把因子排好序的下标"，一个 `order` 同时重排所有列 —— 排一次，处处可用。
- 整数因子（VOI）大量并列 → 并列块内 `stable` 保持时间序（可复现）；一个因子**值**对应一段百分位**区间**。
- 存盘时只留 4000 个等距名次（`np.linspace` 下采样），曲线视觉不变，文件小一万倍。
- dy 的 `shift(-k)` 在 gid（日×午）内做，绝不跨午休/隔夜。

## 五、头尾极端样本的单笔收益（本次新算的）

**原理**：某一段样本的单笔平均收益 = 曲线在那段的**斜率**：

```
mean_dy(q1→q2) = [cum(q2) − cum(q1)] ÷ [(q2−q1) × n]
```

直接从存好的曲线 CSV 上插值即可，不用重跑数据：

```python
c = lambda q: np.interp(q, s.q, s.cum20)          # 任意百分位处的 cum
mean = (c(hi) - c(lo)) / ((hi - lo) * n)          # 该桶的单笔平均（tick）
```

**解读 —— 三个因子三种"尾部性格"**：
- **VOI = 尾部动量型**：5%→1%→0.1% 一路变陡（IC 1.9→2.7→4.1），极端流冲击后价格跟得最狠。全项目唯一明显超过一个往返价差的毛收益 —— 但 0.1% ≈ 每天每边仅约25次，且都发生在行情最猛、成交最难的时刻（对应 fig83 强信号命中率、回测里 +1.9 tick 信号值那一行）。
- **MPB = 饱和型**：印在买/卖价上这个"消息"一旦送达，更极端只是噪声（fig111 的 S 型平顶，换成单笔单位再现）。
- **OIR = 尾部反转型（IM）**：盘口被压到最一边时反而反弹 —— 挂单耗尽/涨跌停微观结构。线性回归会把尾部反号平均进斜率里、永远不告诉你；只有排序视角能看见。⇒ 结论：**尾部行为本身就是因子的一个属性**：VOI 尾部要加仓、MPB 要封顶、IM 的极端 OIR 要反着用。

**诚实备注**：均为毛收益、mid/last 口径混用（VOI/OIR 的 dy 用最新价、MPB 必须用中间价 —— 见 bounce 教训）；桶均值**不受**窗口重叠膨胀影响（每 tick 只贡献一个 dy），是比 cumsum 纵轴更诚实的单位。

## 六、再说一遍意义（一句话版）

> **cumsum 把增量变水位；偏差按 n 累积、噪声按 √n 游走，于是单点看不见的系统性倾向在曲线形状里现形；按因子排序后，这条曲线就是市场对你因子的平均反应函数 —— 斜率是条件均值，谷底是阈值，头尾斜率是极端样本的单笔收益。**

配套警句：纵轴数值被重叠膨胀 ~k 倍、终点是漂移不是信号 —— **看形状、看谷底、看斜率，不看绝对数**。
