# 待问的问题（open questions）

## Q1 · 大脉冲后的反转：均值显著反转，但命中率只有~50%，这是"赔率边际"而非"胜率边际"，怎么理解 / 能不能交易？

**背景（IM 中证1000 股指期货，tick 级，500ms/tick，全历史 2022-07~2026-05）**
- 定义一次"价格脉冲"：回看 n 个 tick 的中间价变动 `back = mid[i] − mid[i−n]`；`|back| > k` 才算触发（k = 阈值，单位指数点）。
- 触发后按方向看未来 x 个 tick 的"符号化收益"：`signed = sign(back) × (mid[i+x] − mid[i])`。`>0` 延续/趋势，`<0` 反转。

**观察到的现象**
1. **存在一条清晰的 hardline**：脉冲超过约 **1.2~1.6 点**（k≈6~8 tick，n≥2）后，`signed` 的**均值翻负（反转）**，且**脉冲越大、反转越深、单调**；统计极显著（t 到 −10 ~ −14），跨 4 年稳定，过滤坏 tick 后基本不变。最强格（如 n=10/k=6点、n=30/k=9点）反转达 **−1.5~−2.2 点**，已超过 ~1.0 点的买卖价差；反转随 x 加深、约 **40 秒**见顶。
2. **但命中率（单次真的反转的比例）几乎从不超过 50%**（全网格 41%~52.6%）。在反转均值最强的小 n 大 k 角落，命中率反而只有 31~44%——**多数脉冲其实是继续**。
3. 也就是说：**负的期望收益完全靠"少数反转的幅度极大"撑起来（幅度不对称 / 肥尾），不是靠"反转更频繁"。**

**想问的**
1. 这是不是已知的微观结构现象？机制是什么——流动性提供者吃单后的反向、动量耗尽 overshoot、还是大单冲击后的均值回复？为什么是"幅度不对称"而不是"频率"？
2. 当边际来自**赔率/幅度**（胜率~50%、赢的时候赢得多）而非胜率时，**扣掉 ~1.0 点价差后还能不能稳定盈利**？该怎么思考仓位 / 风险（肥尾、单边大亏的尾部）？
3. 怎么捕捉最好——入场（脉冲多大才进）、持有期（反转约 40 秒见顶）、止损？这种"靠幅度"的策略对滑点 / 部分成交特别敏感，有没有标准做法？

**已有的图/数据**：`fig_pulse_hardline.png`（hardline + 命中率），`fig_pulse_nkx_byN/byK.png`（n×k×x 全貌），`exp1_nk_clean.csv`（n×k 网格，已过滤坏 tick）。

## Q2 · 反转的胜率不到 50%（约 49%），正期望完全靠幅度——这种"负胜率、正期望"的策略实务上怎么评估和执行？

**背景（同 Q1 设定；价格脉冲 EXP1 与成交量爆发 EXP2 都验证过）**
- 在"期望翻成反转"的代表性阈值处——价格：`|脉冲| > ~1.2~1.6 点`（k≈6~8 tick）；成交量：`最近成交强度 ≥ 2× 平时`（每tick平均量比 spike>2）且持续放量 `t1 ≥ ~20 tick（≈10 秒）`——
- **单次"反转命中率"只有 46~51%**：可靠大样本格最高 ~50.0%，多数 ~49%（**比抛硬币还略差**）。命中率更高的格（51%+）都落在样本几千的小格里、不可信。
- 即：**多数单次其实是"继续"，正期望完全来自少数反转的幅度极大（右尾/肥尾），不是来自方向猜对。**

**想问的**
1. 这种**胜率 < 50%、靠右尾幅度**的策略，怎么评估稳健性？看夏普 / 最大回撤 / 右尾对个别年份的依赖（会不会全靠某几次大反转）？
2. 实务执行：**固定止盈止损**会不会正好把"赚钱靠的那条右尾"切掉？该怎么设计离场（按时间 ~40 秒？按幅度？）？
3. 和**买卖价差 / 滑点 / 部分成交**的相互作用——胜率<50% 时，成本对净期望的侵蚀是不是更致命？这类策略有没有标准的下单/风控做法？
